# 🗂️ Swipe files — Publicidade (o repertório para copiar e adaptar)

**TL;DR:** As gavetas de peças reais que já funcionaram, organizadas por **objetivo**, para o redator/DA puxar e adaptar. Diferente das `tecnicas/` (o *porquê*), o swipe é o *exemplo concreto*. Padrão de organização aprovado por Diego, puxado do corpus `cases-premiados-publicidade-2023-2026`.

## As gavetas (por objetivo)
- **`headlines/`** — headlines vencedoras, indexadas pela estrutura ([[cmo/publicidade/tecnicas/oito-estruturas-de-headline]]).
- **`leads/`** — aberturas de carta de vendas / VSL (por nível de consciência — [[cmo/publicidade/tecnicas/niveis-de-consciencia-schwartz]]).
- **`big-ideas/`** — conceitos que viraram campanha (por forma de construção).
- **`hooks/`** — ganchos de vídeo dos 3 primeiros segundos (cruza com [[cmo/trafego-pago/hook-nos-tres-segundos]]).
- **`ctas/`** — chamadas para ação que converteram.
- **`ofertas/`** — estruturas de oferta/irresistibilidade.

## Formato de cada verbete de swipe
Cada verbete é curto e sempre traz os 3 campos:
```
- **[texto da peça]** — <marca / campanha, ano>
  - Por que funciona: <a técnica que aciona — link para a ficha de tecnicas/>
  - Resultado: <número verificável, quando houver — senão "sem número público">
```

## Regra de entrada
Nada entra bruto (anti collector's fallacy). Cada verbete é processado: identificar a técnica que o explica e linká-la. Sem "por que funciona", não é swipe — é clipping.

## Construção (próximo passo da Fase 2)
Indexar o corpus `cases-premiados` (arquivos 01-07) nessas gavetas, um verbete por peça, começando por headlines e hooks (maior uso no dia a dia).
