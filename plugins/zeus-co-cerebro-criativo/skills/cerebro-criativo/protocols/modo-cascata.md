# Modo CASCATA — Bend → Break → Blend em sequência profunda

> Modo aprofundado. Aplica os 3 vetores **sequencialmente**, cada um usando output do anterior como insumo. Cada etapa explora vertical, não horizontal.
> Uso típico: 1 desafio crítico, 60-90 min de skill, output denso.

## Quando usar Modo Cascata

- Desafio é central / existencial (não tática rápida)
- Diego/LEP quer **profundidade**, não amplitude
- Resultado esperado: **1 conceito profundo defensável** (não 3 ideias pra escolher)
- Pode investir tempo + atenção

## Quando NÃO usar Modo Cascata

- Tempo curto → Modo Solo (mais rápido)
- Skill chamada por outra skill → Modo Suporte
- Já tem ideia, só refinar → Modo Refinamento
- Time travado em senso comum → Modo Absurdo-Controlado primeiro

## Por que sequência Bend → Break → Blend?

Sequência intencional:

1. **Bend primeiro** — recontextualiza. Resultado é variação reframe do óbvio.
2. **Break sobre o output Bend** — desafia constraint da ideia Bend. Resultado é versão radicalizada.
3. **Blend sobre o output Break** — funde a versão Break com domínio distante. Resultado é versão híbrida com propriedade emergente.

Cada etapa **eleva originalidade** (distância semântica acumulada) preservando intenção do briefing.

## Protocolo (12 passos)

### Etapa 0 — Setup
1. Aplica amplificadores (necessidade + restrição + erro)
2. Lê memória contextual + ativa cross-pollination
3. Identifica nodes do slipnet + ativa 3 links FRACO

### Etapa 1 — BEND (passos 4-6)

4. Aplica `operations/bend.md` com 3 técnicas (Analogia Distante, Inversão de Assunção, Put-to-other-uses)
5. Gera 3 ideias Bend
6. **Escolhe 1 Bend** (melhor combinação distância × coerência × atende)

### Etapa 2 — BREAK sobre Bend (passos 7-9)

7. Pega ideia Bend escolhida como **input do Break**
8. Lista enabling constraints DA IDEIA BEND (não do briefing original)
   - "Que assunções a Bend ainda preserva?"
   - "O que ela toma como dado?"
9. Aplica Break (Eliminate / Modify / Reverse) sobre essas constraints
   - Gera 3 variações Break
   - **Escolhe 1 Break** mais ambiciosa que ainda atende necessidade

### Etapa 3 — BLEND sobre Break (passos 10-11)

10. Pega ideia Break escolhida como **Input 1 do Blend**
11. Puxa 3 domínios distantes adicionais (não os já usados na Etapa 1)
    - Para cada, aplica Blend completo (Fauconnier-Turner: Generic Space + Propriedade Emergente)
    - Gera 3 ideias Blend
    - **Escolhe 1 Blend** final

### Etapa 4 — Síntese (passo 12)

12. **Conceito final** = output do Blend
    - Apresenta evolução completa: Briefing → Bend → Break → Blend
    - Cada etapa tem rationale neuro explícito
    - Conceito final tem:
      - Distância semântica acumulada (geralmente 4-5/5)
      - Coerência verificada por Fauconnier-Turner 5 princípios
      - 3 propriedades emergentes (1 por etapa)
      - Defesa em camadas (cada etapa adiciona força)

## Tempo esperado

- 15-30 min de skill (geração + 3 escolhas + stress-test em cada etapa)
- Output mais denso e profundo que Solo

## Exemplo de invocação

> "cerebro-criativo, conceito completo Crazyflips modo Cascata"

→ Skill retorna 1 conceito final profundo com toda evolução documentada.

## Variação — Cascata invertida

Em casos raros, ordem pode ser invertida (Blend → Bend → Break) se briefing pede **combinação primeiro**. Skill avalia e escolhe ordem se Diego/LEP não especificar.
