# Léxico cross-domain — JOGOS

> Vocabulário e mecânicas de jogos (game design) como matéria-prima pra Bend/Blend. Game design é literatura rica em **engagement loops** e **sistemas de recompensa**.

## Dimensões centrais

- **Mecânica** (regra básica de interação)
- **Dinâmica** (comportamento emergente da mecânica)
- **Estética** (sentimento que o jogo evoca)
- **Loop** (ciclo de ação-recompensa)
- **Progressão** (XP, level, unlock)
- **Multiplayer** (solo, cooperativo, competitivo)
- **Tempo** (turn-based, real-time, async)
- **Estado** (save/load, permadeath)

## Conceitos importáveis

| Conceito | O que carrega | Pra que serve fora |
|---|---|---|
| **XP / Level up** | Acumula → avança | Loyalty program, status tier |
| **Quest / Mission** | Objetivo discreto recompensado | Task completion, checklist gamificada |
| **Boss fight** | Desafio principal de fase | Decisão crítica recompensada |
| **Easter egg** | Surpresa escondida para fãs | Detalhe oculto para power users |
| **Loot box** | Recompensa aleatória paga | Surprise box / mystery box |
| **Permadeath** | Erro = morte permanente | Stakes reais (ex: investidor) |
| **Save point** | Checkpoint pra restaurar | Auto-save state, undo |
| **NPC** | Personagem não-jogador no mundo | Chat bot ambiental |
| **Tutorial** | Aprendizagem assistida inicial | Onboarding gamificado |
| **Skill tree** | Caminhos de upgrade ramificados | Roadmap de carreira / customização |
| **Inventário** | Coleção de itens carregáveis | Wishlist, cart, dashboard |
| **Crafting** | Combina items pra criar | Modular product builder |
| **PvP / PvE** | Player vs Player / Environment | Competição vs colaboração modes |
| **Guild / Clan** | Time formal de jogadores | Comunidade VIP / squad |
| **Lobby** | Sala de espera pré-jogo | Pre-event hub |
| **Matchmaking** | Pareamento por nível | Algoritmo de match (Tinder!) |
| **Leaderboard** | Ranking visível | Top customer wall |
| **Achievement** | Conquista desbloqueada visível | Badge / certificado |
| **Daily reward** | Recompensa por login diário | Streak (Duolingo!) |
| **Battle pass** | Tier season com unlocks pagos | Subscription temporal |
| **Microtransaction** | Compra pequena dentro do jogo | Add-on em SaaS |
| **Speedrun** | Completar o mais rápido possível | Power user fast lane |
| **Modding** | Modificar o jogo | Plugin / extensão community-made |
| **Sandbox** | Sem objetivo, livre exploração | Free-form tool (Notion, Figma) |
| **Roguelike** | Cada run diferente, dificuldade alta | Random generative product |
| **Souls-like** | Dificuldade extrema com aprendizado | Produto pra "hardcore users" |
| **Battle royale** | Many vs many, last standing wins | Competição mass |
| **Cooperativo (Co-op)** | Time vence junto contra ambiente | Multi-stakeholder buying |
| **Asynchronous multiplayer** | Joga sem estar online | Async work / contribuição |

## Mecânicas de engajamento (Bartle taxonomy)

| Tipo | Motivação | Aplicação |
|---|---|---|
| **Achiever** | Vencer, completar, otimizar | Gamification tradicional (XP, leaderboard) |
| **Explorer** | Descobrir, entender, mapear | Hidden features, deep docs |
| **Socializer** | Conectar, conversar | Comunidade, fórum |
| **Killer** | Dominar outros | PvP competitivo (anti-padrão pra muitas marcas) |

## Conceitos de game feel

| Conceito | Origem | Aplicação |
|---|---|---|
| **Juice** | Feedback visual/sonoro exagerado | UI delight (Apple, Spotify Wrapped) |
| **Flow state** (Csikszentmihalyi) | Desafio ↔ habilidade balanceado | Difficulty curve em UX |
| **Coyote time** | Pulinho extra fora da plataforma (perdão) | UX que perdoa erros menores |
| **Game feel** (Steve Swink) | Sensação tátil do controle | Latência, haptic feedback em produto |
| **Compulsion loop** | Ciclo que vicia | Engagement (com ética!) |
| **Skinner box** | Recompensa intermitente variável | Random reward (alerta ético) |

## Frameworks de game design

| Framework | Origem | Aplicação |
|---|---|---|
| **MDA (Mechanics, Dynamics, Aesthetics)** | Hunicke et al. | Estrutura de design de experiência |
| **8 Kinds of Fun** (LeBlanc) | Sensation, Fantasy, Narrative, Challenge, Fellowship, Discovery, Expression, Submission | Mapear tipo de experiência |
| **The 4 keys to fun** (Lazzaro) | Hard fun, Easy fun, Serious fun, People fun | Categorias de prazer |
| **Hooked model** (Eyal) | Trigger → Action → Variable Reward → Investment | Habit-forming product |
| **Octalysis** (Yu-kai Chou) | 8 motivadores core (epic meaning, ownership, etc) | Análise de gamification |
| **The Art of Game Design (Schell)** | Lens-based design — 100+ lenses | Toolkit de questionamento |

## Como skill usa

- Quando briefing precisa **engagement repetido** (loop)
- Quando precisa **estrutura de progressão** (XP, tier, badge)
- Quando precisa **social mechanic** (multiplayer, leaderboard, guild)
- Quando precisa **feedback satisfação** (juice, achievement)
- Para **produto SaaS** com elementos gamificados
- Para **comunidade** com hierarquia clara
- Para **naming**: Quest, Loot, Save, Guild, Boss, Tier, Pixel, Loop, Pivot

## Avisos

- Gamificação aplicada errado vira manipulativo. Cuidado com Skinner box / variable reward em domínios sérios.
- Bartle "Killer" type pode polarizar — use só onde competição é o produto.
