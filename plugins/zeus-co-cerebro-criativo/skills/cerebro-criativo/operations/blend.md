# BLEND — Protocolo operacional (Misturar / Combinação)

> Equivale a **Combinatorial Creativity** (Boden) — combina ideias familiares de formas não-familiares.
> SCAMPER: **Combine** (operação única).
> Fundação cognitiva: Fauconnier-Turner Conceptual Blending Theory — 4 espaços (Input 1, Input 2, Generic, Blended).

## Quando usar Blend

- Briefing pede algo **novo** sem que se queira quebrar regras
- Categoria nova de produto / mercado
- Naming (Blend linguístico — Liquid Death, Net+flix, Insta+gram)
- Posicionamento de marca (Blend de 2 territórios)
- Conceito de campanha que junta 2 universos
- Lançamento que combina formatos (música + jogo, série + comunidade)
- Modelo de negócio que combina 2 modelos clássicos (SaaS + Marketplace, Assinatura + Físico)

## Quando NÃO usar Blend

- Briefing pede reframe — use Bend
- Briefing pede disrupção — use Break
- Briefing já tem solução e quer só refinar — use Refinamento

## Protocolo (10 passos) — Aplica framework Fauconnier-Turner direto

### 1. Recebe briefing e identifica desafio
- O que precisa existir e não existe?
- Que problema isso resolve?

### 2. Identifica Input Space 1 — domínio óbvio do desafio
Domínio "natural" pra esse problema. Ex. desafio "marca de água" → Input 1 = "água mineral".

### 3. Puxa 3-5 candidatos para Input Space 2 — domínios distantes
Do `corpus/lexicons/`. **Quanto mais distante, maior potencial criativo** (Beaty/SemDis).

Para "água mineral" candidatos distantes:
- música punk
- esporte radical
- religião apocalíptica  
- moda streetwear
- jogo competitivo

### 4. Pra cada par (Input 1, Input 2) — identifica Generic Space
**Generic Space é o que torna o blend possível.** Sem estrutura abstrata compartilhada, blend é incoerente.

Exemplo (Liquid Death real):
- Input 1: água mineral → função hidratação saudável
- Input 2: cerveja punk → estética transgressora, comunidade
- Generic Space: "**bebida com atitude cultural forte**" — propriedade compartilhada abstrata

### 5. Constrói Blended Space — propriedades emergentes
**A propriedade emergente é o ouro.** Sem ela, blend é só justaposição.

Liquid Death blend:
- Visual: latas pretas com caveira
- Tom de voz: punk/metal/horror
- Posicionamento: "morrer de sede" → invertido → "death to thirst"
- Sustentabilidade: contra plástico (entra do punk anti-corporativo)
- Emergente: bebida sem álcool com cred cultural de bebida alcoólica → **comunidade straight edge premium**

### 6. Aplica 5 princípios de otimização (Fauconnier-Turner)
Pra cada blend candidato:
- ✅ **Integration**: blend tem coerência interna?
- ✅ **Web**: relações dos inputs continuam visíveis?
- ✅ **Unpacking**: dá pra reconstruir Input 1 e Input 2 a partir do blend?
- ✅ **Topology**: relações estruturais preservadas?
- ✅ **Good Reason**: cada elemento tem motivo de estar?

### 7. Gera 3-5 blends mais fortes
Cada um carrega:
- **Input 1 + Input 2**
- **Generic Space**
- **Blended Space**
- **Propriedade emergente**
- **Distância (Input 1 ↔ Input 2)** — proxy de originalidade

### 8. Stress-test neuro
- Necessidade atendida?
- Restrição respeitada?
- Errors óbvios? (parecer forçado, ser ilegível)

### 9. Rankeia
Top 3 por **distância × coerência × propriedade emergente forte × atende necessidade**.

### 10. Rationale neuro inline + Próximos Movimentos
"Blend de [X] + [Y]. Generic Space: [Z]. Propriedade emergente: [W]. Distância 4/5."

## Exemplos resolvidos

### Exemplo 1 — Briefing: "Conceito Crazyflips (NFT + audiovisual)"

**Input 1 (óbvio):** NFT colecionável

**Candidatos Input 2 distantes:**
- Religião sincrética brasileira (Umbanda, Candomblé)
- Manga japonês (estética + serialização)
- Esporte amador comunitário
- Música indie nicho (cassettes de fanzine)

**Blend A (Religião sincrética + NFT)**:
- Input 1: NFT (digital, escasso, transferível, especulativo)
- Input 2: Religião sincrética BR (panteão diverso, oralidade, comunidade local, ritual)
- Generic: "sistema de identidade simbólica com hierarquia interna"
- Blend: NFTs como "entidades" do panteão Crazyflips, cada coleção é um "axé", drops são "festas", oração-coleta de XP
- Propriedade emergente: NFT com **devoção** (não só especulação)
- Distância 5/5

**Blend B (Manga + audiovisual)**:
- Input 1: NFT + audiovisual original
- Input 2: Manga seriado japonês (capítulos semanais, cliffhanger, edição coletada)
- Generic: "narrativa serializada com objetos colecionáveis"
- Blend: drop semanal de "capítulo" audiovisual, coleção completa libera animação especial
- Propriedade emergente: NFT que ganha valor **narrativo**, não só econômico
- Distância 4/5

### Exemplo 2 — Briefing: "Naming via Blend pra 2ndStreet"

**Input 1:** streetwear celebrity / cultura urbana
**Input 2 distantes:** religião, esporte, gastronomia, música, mitologia

- "ALTAR" (Religião) — peças "altar" do hype, ritualização de lookbook → SACRO STREET
- "BACKSTAGE" (Música) — peças "pre-show" exclusivas → SHOWROOM (cliché — rejeitar)
- "RECEITA" (Gastronomia) — cada peça tem "receita" de proveniência → FORMULA
- "JOGO" (Esporte) — cada peça é "rodada", colecionar = campeonato → RACK
- "PANTEÃO" (Mitologia) — coleção de "deuses" celebrity → PANTHEON

## Erros comuns em Blend

1. **Input 2 muito próximo de Input 1** — vira variação, não blend. "Água + suco" não é blend interessante.
2. **Sem propriedade emergente** — apenas justapõe traits. Não emerge nada novo.
3. **Forçar todos os 5 princípios de otimização equivalente** — alguns blends ganham com Topology forte, outros com Web. Aceita trade-offs.
4. **Pular Generic Space** — sem estrutura abstrata compartilhada, blend é colagem aleatória, não conceito.
5. **Blend de 3+ inputs (Multi-Blend)** — possível, mas mais difícil de manter coerência. Skill prefere 2 inputs primeiro, depois multi se Diego pedir.
