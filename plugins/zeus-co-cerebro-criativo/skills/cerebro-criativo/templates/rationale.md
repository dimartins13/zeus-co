# Template — Rationale neuro (defesa por ideia)

> Cada ideia gerada pela skill carrega rationale neuro defendível em 1-3 frases. Este template estrutura como construir essa defesa.

## Princípio

Rationale neuro = explicação **por qual mecanismo cognitivo** a ideia foi gerada. Não é "ficou legal porque combina X". É "esta ideia é Bend importando estrutura de [domínio Y] pra [domínio Z], ativando node distante do slipnet via link FRACO".

Rationale tem 3 funções:
1. **Defesa intelectual** — quando alguém pergunta "de onde tirou isso?"
2. **Reprodutibilidade** — outra skill/pessoa consegue replicar o método
3. **Calibração futura** — quando ideia funcionar, sabemos PORQUE funcionou

## Estrutura do rationale por vetor

### Rationale BEND

```
Bend: [conceito do briefing] dobrado via [estrutura/lógica de domínio distante Z].
Mapping: [3 elementos do domínio Z aplicados ao conceito].
Slipnet path: [link FRACO ∙∙∙ ativado].
Propriedade emergente: [o que nasceu da dobra].
```

#### Exemplo
> Bend: "campanha TarjaPreta" dobrada via estrutura de **liturgia católica**.
> Mapping: rito de passagem (onboarding como batismo), confissão semanal (check-in), congregação (comunidade).
> Slipnet path: LANÇAMENTO ∙∙∙ rito de passagem (religião).
> Propriedade emergente: app de saúde mental com **devoção** (não consumo passivo) — categoria nova.

### Rationale BREAK

```
Break: do enabling constraint [X] (assunção fundadora removida/modificada/invertida).
Espaço novo: [como o novo espaço funciona, regras R'].
Precedente cross-setor: [case análogo que quebrou constraint similar].
Risco principal: [modo de falha previsível].
```

#### Exemplo
> Break: do enabling constraint **"cassino tem prêmio financeiro"** (eliminado).
> Espaço novo: iGaming onde valor é **cultural-cognitivo** (ranking, reconhecimento), não monetário.
> Precedente cross-setor: Crossfit Games (esporte amador profissionalizado por reconhecimento, não dinheiro inicial).
> Risco principal: usuários acostumados a azar monetário podem rejeitar (probabilidade média) — mitigação via segmentação inicial pra perfil "competitivo cognitivo" (xadrez, esports).

### Rationale BLEND

```
Blend de:
  Input 1: [conceito A]
  Input 2: [conceito B — domínio distante]
Generic Space: [estrutura abstrata compartilhada que torna fusão possível].
Blended Space: [como nova entidade opera].
Propriedade emergente: [o que nasceu que nenhum input sozinho tinha].
Princípio FT mais forte: [Integration / Web / Unpacking / Topology / Good Reason].
```

#### Exemplo
> Blend de:
>   Input 1: marketplace streetwear celebrity
>   Input 2: ritual de **panteão religioso sincrético BR** (Umbanda)
> Generic Space: sistema de identidade simbólica com hierarquia interna + relação devoção-objeto.
> Blended Space: 2ndStreet como "axé" cultural — cada peça é "entidade" do panteão, drops são "festas", colecionar é "iniciação".
> Propriedade emergente: revenda com **devoção** (não especulação) — diferencia de ThredUp/Vestiaire.
> Princípio FT mais forte: Topology (estrutura hierarquia panteão preservada em hierarquia colecionar).

## Rationale composto (em Modo Cascata)

Quando ideia veio de Cascata Bend → Break → Blend, rationale tem 3 camadas:

```
ETAPA 1 — Bend [conceito base] via [domínio Z]
  ↓
ETAPA 2 — Break do enabling constraint [Y] da Bend
  ↓
ETAPA 3 — Blend do output Break + [domínio W]

Resultado final: [conceito com 3 propriedades emergentes acumuladas]
Distância semântica acumulada: 5/5
```

## Rationale em Modo Suporte (API mode)

Quando skill é chamada por outra skill, rationale é **mais curto e tipado**:

```
Vetor: [X]
Domínio distante: [Y]
Distância: X/5
```

(1 linha, sem aula, suficiente pra caller saber de onde veio.)

## Anti-padrões em rationale

### ❌ Rationale vibe-only
> "Ficou legal porque é diferente."

→ Não explica mecanismo, irreprodutível, indefensável.

### ❌ Rationale teórico sem aterrissar
> "Aplicamos a teoria de blending conceitual de Fauconnier-Turner em conjunção com a taxonomia Bodeniana de criatividade transformacional..."

→ Aula, não defesa. Diego precisa apresentar pro Akkari, não pra journal acadêmico.

### ❌ Rationale auto-elogioso
> "Esta ideia é genial porque combina X com Y de forma criativa."

→ Sem rationale defendível, só afirmação.

### ✅ Rationale bom
> "Bend importando ritual de combate (UFC round) pra check-in de saúde mental. Slipnet path: PRODUTO ∙∙∙ jogada esportiva. Propriedade emergente: confronto sadio com demônios internos como esporte, não terapia paciente."

→ Mecanismo claro, mapping explícito, propriedade emergente identificada, 3 linhas.

## Princípios

1. **Curto** — 3-5 linhas, não 3 parágrafos
2. **Mecanismo explícito** — qual vetor, qual mapping, qual slipnet path
3. **Aterrissado** — usar caso ou metáfora concreta, não só teoria
4. **Defensável** — Diego consegue defender pro Akkari/board/investidor
5. **Reprodutível** — outra pessoa/skill conseguiria gerar ideia similar com mesmo método

## Uso em apresentações externas

Quando Diego/CCO vai apresentar conceito derivado da skill pra terceiros, **omite o vocabulário neuro** (slipnet, Fauconnier, distance forçada) e usa **versão narrativa**:

### Rationale interno (skill → Diego)
> Blend de marketplace + panteão Umbanda. Generic: sistema de identidade simbólica hierárquica. Distância 5/5.

### Versão narrativa externa (Diego → Akkari)
> "Quero que a 2ndStreet não seja revenda de roupa. Quero que seja uma cultura — onde cada peça é uma entidade, cada drop é uma festa, colecionar é entrar pra uma tribo. Como uma religião urbana com Neymar e você como sacerdotes culturais. ThredUp e Vestiaire vendem roupa usada. A gente vende devoção."

Mesma ideia. Rationale interno explica o método. Versão narrativa apresenta o resultado.
