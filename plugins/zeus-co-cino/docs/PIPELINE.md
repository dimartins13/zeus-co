# PIPELINE — CINO Office

> CINO inventa o futuro. CTO constrói. CMO vende.
> Foco: H2 (adjacent) + H3 (transformational) do Three Horizons.

---

## Pipeline (7 fases)

### Fase 0 — Descoberta Interna

### Fase 1 — Innovation strategy
Owner: `cino` (chief)
- Three Horizons: H1 (core) 70% / H2 (adjacent) 20% / H3 (transformational) 10%
- Áreas de aposta (ex: dope street → AR try-on; TarjaPreta → wearables; etc.)
- Output: `_Areas/CINO/innovation-strategy.md`

### Fase 2 — Research aplicada (discovery)
Owner: `cino-research`
- User research profundo
- Jobs-to-be-done framework
- Behavioral insights
- Output: `_Areas/CINO/research/<topic>.md`

### Fase 3 — Tech scouting
Owner: `cino-tech-scouting`
- Tendências tech setor
- Startups emergentes
- Patentes recentes
- Conferências importantes (CES, SXSW, NRF)
- Output: `_Areas/CINO/tech-radar.md` (atualizado mensal)

### Fase 4 — Experimentation (lean)
Owner: `cino-experimentation`
- Hypothesis-driven experiments
- MVP minimal pra testar
- Kill metrics claras
- Output: `_Areas/CINO/experiments/<exp>.md`

### Fase 5 — Open innovation
Owner: `cino-open-innovation`
- Hackathons (interno + externo)
- Parcerias com universidades (USP, Insper, etc.)
- Aceleradoras + venture studios
- Output: `_Areas/CINO/partnerships.md`

### Fase 6 — Stage-gate (matar ou escalar)
Owner: `cino` + cross `ceo-estrategia` + `cfo`
- Cada experimento passa por gates
- Gate 1 (research): aprende algo novo? → Fase 4
- Gate 2 (MVP): customer paga? → produto
- Gate 3 (produto): escala? → vira nova vertical / spin-off

### Fase 7 — Handoff
Owner: `cino` → `cto-pm` (se vira produto) ou `entrepreneurship` (se vira empresa nova)
- Documentação completa
- Team transfer
- Output: handoff memo

---

## Princípio inviolável

**Mais ideias mortas > poucas ideias bem-sucedidas.** Inovação é jogo de números. Maioria das ideias morre. As que sobrevivem pagam o jogo todo.
