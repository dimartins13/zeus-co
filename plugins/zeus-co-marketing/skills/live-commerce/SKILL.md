# Live Commerce — Vendas Ao Vivo Digitais

## Identidade

Sou responsável pela **camada de vendas em tempo real digital** — onde host apresenta produto, audiência responde no chat, compra no link em segundos. Distinto de `live-marketing` (eventos físicos BTL/drops presenciais) — eu sou **vídeo + chat + checkout, sem evento físico necessário**.

## Princípio inviolável

**Live commerce é VAREJO em formato vídeo. Não é marketing.** Métricas que importam: GMV por live, AOV (Average Order Value), conversão de viewers em buyers, watch time. NÃO é "reach" ou "engagement". Se você vai medir live commerce como awareness, está fazendo errado — use live-marketing ou social tradicional.

Segundo princípio: **host quality + product fit + frequency** são os 3 drivers. Falhar em 1 dos 3 = live fracassada. Host genérico vende 5%; host certo vende 50%.

**Fase 7k** (Execução criativa multi-formato — NOVA sub-fase). Canal de conversão direto, alta intensidade operacional.

## Implementação (setup técnico)

### 1. Setup de conta + integração catálogo
- **TikTok Shop:** abrir TikTok Shop seller account → catálogo CSV ou via Shopify/integration → ativar Live feature (precisa >1000 followers + verificação)
- **Instagram Live Shopping:** Meta Commerce Manager → catálogo conectado → tag produtos ao vivo
- **Shopee Live:** Shopee Seller Center → ativar Live → catálogo já no Shopee
- **Mercado Livre Live:** Centro de Vendedor → Live Streaming
- **Magalu Live:** Magalu Marketplace seller → Live activation manual com Magalu time

### 2. Stack técnico
- **Streaming software:** OBS (free, profissional) ou Streamlabs (mais simples)
- **Câmera:** DSLR/Mirrorless via capture card OR webcam premium (Logitech Brio 4K)
- **Microfone:** **CRÍTICO**. Lavalier (Rode Wireless GO) ou condensador (Shure SM7B + interface)
- **Iluminação:** softbox + ring light (não-negociável)
- **Cenário:** mínimo 2 layers (foreground + background) + branding visual
- **Encoder:** stream em 1080p 30fps minimum, 60fps idealmente
- **Internet:** upload 10Mbps+ estável (cabeada, não wifi)
- **Backup stream:** sempre ter PC/notebook secundário pronto

### 3. Pre-live operacional (T-7 a T-1)
- **T-7 dias:** anúncio + thumbnail + agendamento + push pra base
- **T-3 dias:** post de teaser social + email pra base CRM
- **T-1 dia:** lembrete push notifications + stories countdown
- **T-2 horas:** ensaio com produtos + checagem stream tech
- **T-30 min:** stream test (going live nas plataformas)

### 4. Estrutura de live (template canônico)
- **00:00-02:00** Hook + apresentação host + agenda da live (o que vão ver)
- **02:00-05:00** Cold open com PRIMEIRO produto + oferta exclusiva live (urgência)
- **05:00-50:00** Loop de produtos (cada produto: showcase 3min + Q&A chat 2min)
- **50:00-55:00** Recap + drop adicional + urgência final
- **55:00-60:00** CTA pra próxima live + agradecimento

### 5. Tracking
- **GMV per live** (Gross Merchandise Value)
- **Unique viewers**
- **Peak concurrent**
- **Watch time average**
- **Conversion rate** (viewers que compraram)
- **AOV** (Average Order Value)
- **Returning viewers** (% que vieram de live anterior)
- **Chat engagement** (msgs/minuto)

## Gestão (operação contínua)

### Host management
**Critério-chave de seleção:**
1. **Comfort com produto** (saber explicar sem ler script)
2. **Energia consistente** (live de 60min cansa — host precisa aguentar)
3. **Capacidade de responder chat em tempo real** (multitasking)
4. **Brand fit** (precisa parecer da audiência)
5. **Disponibilidade recorrente** (live precisa cadência semanal+)

### Cadência recomendada
- **Início:** 1 live/semana, mesmo horário, mesmo dia
- **Escala:** 2-3 lives/semana, hosts variados
- **Especiais:** lives mensais com convidados (celebridade, expert)
- **Drops:** lives sob demanda quando produto novo

### Calendário editorial de lives
| Tipo | Frequência | Duração | Foco |
|---|---|---|---|
| Live regular semanal | 1× sem | 60min | Catálogo amplo + Q&A |
| Drop ao vivo | Sob demanda | 30-45min | Produto único, urgência alta |
| Live colaborativa (creator) | 1× mês | 60-90min | Audience hijack + UGC |
| Live evento (Black Friday, datas) | 4× ano | 90-180min | Variedade + entretenimento |

### KPIs (saudáveis pra benchmark BR 2026)
| Métrica | Saudável | Excelente |
|---|---|---|
| **Conversion (viewer → buyer)** | 1-3% | 5-10% |
| **AOV** | R$ 80-200 (mass) / R$ 300-800 (premium) | — |
| **Watch time avg** | 6-12min | 15-25min |
| **Peak concurrent** | 50-500 | 2k-10k+ |
| **GMV/hora** | R$ 5k-30k | R$ 50k-300k |

### Otimização (loop semanal)
- Quais produtos venderam mais → repetir + variação
- Quais momentos teve queda de viewers → editar template
- Que perguntas repetiram no chat → FAQ pra próxima live
- Qual host performou melhor com qual categoria → matching

## Heurísticas operacionais

- **Primeiros 5min decidem audience retention 60min** — abertura precisa ser hook real
- **Frequency cap psicológico:** mesma audience pode aguentar live 2-3× semana max, não mais
- **Vendas concentram nos 3 últimos minutos** — sempre fazer "última oferta" no fim
- **Cross-promotion entre lives** — fim de uma live = teaser próxima
- **Re-watch é OURO:** edita clips das melhores partes pra post + Reels — alimenta funil pra próxima
- **Discount mecânico funciona melhor:** "primeiros 10 que comentarem X ganham 20%" > genérico
- **Quanto mais ao vivo, melhor — mas qualidade > quantidade.** 1 live boa/sem > 5 lives genéricas/sem

## Heurísticas BR

- **TikTok Shop BR lançou 2024, explodiu 2025** — momentum único. Quem entra cedo ganha algoritmo.
- **Magalu Live e Mercado Livre Live** têm audience massa, mas algoritmo dá menos push pra new sellers — começar com paid push.
- **Horário canônico BR:** 19h-22h dias úteis, 14h-18h fim de semana.
- **Falar PT-BR coloquial natural** — gírias regionais funcionam, "voilá" não.
- **CDC garantia + arrependimento 7 dias** — disclosure no live + termos atualizados
- **CDC publicidade enganosa** — promessas só baseadas em fato (não exagerar "queima de estoque" se não é)
- **CONAR + publicidade infantil** — public alvo <12 anos restrições especiais

## ⚙️ Channel Skill — não-LEP

Esta é uma **Channel Skill** (execution skill de canal/tática específica), não um LEP.

**Diferença operacional:**
- LEPs (`cmo`, `cmo-branding`, etc) têm identidade, pipeline próprio, orquestram outras
- Channel Skills (esta) são **ferramentas táticas** despachadas pelo `cmo` ou `marketing-orquestrador`
